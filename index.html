「Webコンテンツ作成」の成果

本テーマについて

本テーマでは、学習目標を元に第1回から第6回まの授業で、最終的に個人のプロジェクト課題を完成させることが目的です。この授業を通して、HTMLやCSS。JavaScriptの基本的な操作方法を学び、Web制作についてのノウハウを学習します。

プロジェクト課題について

今回のプロジェクト課題では、グルメ情報を選択しました。検索キーに探したい「ジャンル」または「キー」を入力し、検索ボタンを押すと条件に合った店舗が羅列されます。また、もし検索キーに該当しない。入力が間違えている際はエラー文を表示する様にしました。このプログラムを作成するにあたって、画像がある場合はアイコンにそれを利用し、店名や表示するトピックを見出し表記にしたり、全体的に目に優しい配色や形にするなど。CSSなどを活用して見やすい様に工夫しました。

  • プロジェクト課題で作成したページ： project.html

各授業回の学習事項

第1回 WWW と HTML

本講義にあたっての事前準備とWWWとHTMLについての基本的なことを学びました。「タグ」や「内容」「要素」などの文書を作るにあたって基本的になる形や、それに付随する「属性」を学習しました。HTML文書の構成では必ず初めに作らなければならない!DOCTYPE htmlやWebページの情報のうち表示するしない部分を表すheadやbodyなどの、HTML文章を制作する上で基本となってくるものも学べました。また、HTML要素リファレンスやHTMLのデバッグにて、今後のWeb作成に必要不可欠なサイトの紹介もありました。

  • 課題で作成したページ： 演習1-4: HTMLのさまざまな要素

第2回 CSS による Web ページの装飾

今回の講義内容は、主にCSSによるWebページの装飾方法でした。中でも押さえておきたい内容が、最初に紹介された、link lerは装飾したいWebページと実際に編集を行うCSSとを繋ぐための重要なコマンドでした。また、fontやbackgroundなどのプロパティとその値などを学びました。そして、それらを検索し実際に試行できるサイトの紹介もありました。

  • 課題で作成したページ: 演習2-3: ウェブブラウザ説明ページの装飾

第3回 JavaScript の基本

今回の講義では、javaScriptについての基本的な内容について学習しました。過去プログラム1,2やそれら以外で主に使用していた言語であるJavaとの違いを実際に実感しながら感じれたと思います。また、jsの編集やwebで使えるデバッガーなどの操作も含め、今後本講義の最終的な目標であるproject.htmlを完成させるためにとても重要な事項を学べました。しかし、使い慣れていないせいかとても理解するのに時間がかかりそうでした。特に、言語が違うので今までやっていたJavaの癖でプログラムを間違えることが多々ありました。構文も全く違うのでこれらに注意を払いながら自らのものにしたいと思います。

  • 課題で作成したページ： 演習3-6: オブジェクトの参照

第4回 DOM と JavaScript

今回の講義では、主に DOM について学習しました。おそらく、どの検索サイトや Web ページの装飾にもあるであろうボタンの作用を、この DOM を使って実現できることを学びました。今回の DOM の講義を受けて感じたのは、ただボタンを配置し、そのボタンに機能を持たせるだけでも、それに使うプログラムの種類や量がとても多く、大変だということです。実際に手を動してみて、そのことを実感しました。ラベルを作り、テキストボックスとボタンを配置し、それらに使われる `input` 要素や、配置したボタンを実行可能にするためのイベントハンドラなど、多くの要素を組み合わせて実装する必要がありました。今回は講義の内容としては第1・2・3回よりも少なかったですが、内容の密度は非常に濃かったと感じています。これまでの講義を通して、普段自分が使っている Web サイトの中身を、少しずつ自分でも作れるようになってきている実感があります。そのため、JavaScript へのモチベーションも高まっていると感じています。とても難しい内容ではありますが、自分のものにできるよう、今後もさらに学習を進めていきたいと思いました。

  • 課題で作成したページ: 演習4-10: 足し算ページ

第5回 イベント処理

今回の講義では、前回に続き DOM についてさらに深く学習しました。要素の新規作成、要素の編集や要素の変更などがありました。要素を配置する際に使うbeforeendやafterendなどが少し覚えるのに苦労しました。また、最後の課題で行ったjsにcssを反映させるために行なった装飾が今回一番難しかったです。参考資料のようにすればもちろん簡単にできるのでしょうけど、前に自分が作成したものをまた引っ張り出してそれにしっかり合わせてプログラムを行うのはとても骨が折れました。「自分が作ったのになぜ、、、」とつくずく自分の記憶力や吸収能力の低さに落胆します。ですが、もう今講義も大詰めなので、最後までやれるだけがんばりつつ少しでも自分のものにできるよう頑張りたいと思います。

  • 課題で作成したページ: 課題5-2: index.html を編集しよう

第6回 総括と参考資料

今回の授業で習ったWeb制作において、全体的にとても難しく、慣れるまでがとても大変でした。大学に入り最初に習ったJava言語が身についているので、そっちの言語にどうしても引っ張られ、授業の初めの頃は言語に慣れず大変苦労しました。第一回で学んだ、HTMLについては、HTMLの特殊文字や文書のコメントが独特だと感じ、特に、if文で比較したい際に癖でJavaのやり方で書いてしまうことが多々ありました。第二回ではWeb制作において文字や枠どりなどの見た目を編集できるCSSを学びました。いままでいつも使っているWebページの文字フォントやサイズ、間取り等は全て一つのページで全部を編集できると思っていました。しかし、この回の授業でCSSというスタイルシート言語によって編集されていることを学びました。CSSではプロパティの多さにとても驚きました。後述するproject課題のCSSでは全体的に見やすい色で目に優しく、わかりやすい様に工夫しました。唯一この授業で比較的やりやすく簡単な部分でした。第三回ではJavaScriptを学びましたが、今改めて振り返ってみると、Javaより書くことが少なかったり、コンパイルが不要だったり、文法が比較的緩かったりとそもそも目的の違う言語ではありますがJavaScriptのほうが簡単だと後になって身にしみました。しかし、第四回と第五回のDOMが絡んでくると表情が曇り始めました。イベント処理が組み合わさったあたりで演習や課題、project課題の方でもエラーを吐きまくりました。ですがおそらくそれは自分の理解不足のせいなのでとても不甲斐ない結果でした。第六回では最終目標であるproject課題を完成させるため、3つのことをやりましたが、検索キーの設定でとても手こずりました。自分の選択したproject課題はグルメ情報であったため、検索キーは1つだけでしたが、検索窓にカフェやGなにがしと入れた際にヒットするようにURLを編集するのが大変でした。どちらでも検索にヒットするために配列とif文を使ってジャンルとキーを対応させました。また、どちらにも該当しない文字が入力された際にエラー文とジャンル名・キーの一覧を載せる様にしました。授業全体を通して少しはWeb制作に関する知識とスキルを会得できたと思いますじつは他の授業でもWeb制作をする機会がありました。そして今回の授業と合わせて意外と制作が楽しく、面白しろくありました。将来Web制作関連の仕事も悪くないと思いました。
